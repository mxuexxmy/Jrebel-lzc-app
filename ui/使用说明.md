# JRebel 使用说明

本文说明如何在 IDEA 中安装、激活 JRebel，并使用本项目的激活服务进行离线激活与热部署开发。

---

## 一、简介

### JRebel 是什么？

JRebel 是一款 **Java 热部署插件**，可在修改代码后**无需重启应用**即让更改实时生效。它工作在 JVM 层，支持 Spring、Tomcat、Hibernate 等主流框架，能显著提升开发效率。

### 与本项目的关系

- **JRebel Application**（本仓库的 Web 页）：展示基于 JRebel 技术的应用说明与状态。
- **激活服务**：使用 [yu-xiaoyao/jrebel-license-active-server](https://github.com/yu-xiaoyao/jrebel-license-active-server) 作为 JRebel 许可证激活服务器，支持**离线激活**。

### 插件对比（简要）

| 插件             | 支持范围       | 热部署能力           |
|------------------|----------------|----------------------|
| **JRebel**       | 全栈           | 类/方法级实时更新    |
| Spring DevTools  | 仅 Spring Boot | 有限（不含静态资源） |

## 二、激活服务器使用（jrebel-license-active-server）

激活前需要先让「激活服务」在本地或本机可访问的地址运行。

### 方式一：本地运行（推荐）

1. **下载**  
   打开 [Releases](https://github.com/yu-xiaoyao/jrebel-license-active-server/releases)，选择对应系统：
   - Windows 64 位：`jrebel-license-active-server-windows_amd64.exe`
   - macOS：`jrebel-license-active-server-darwin_amd64`（Intel）或 `darwin_arm64`（Apple Silicon）
   - Linux：`jrebel-license-active-server-linux_amd64`

2. **运行**  
   - Windows：双击 `.exe` 或在命令行运行。  
   - Mac/Linux：解压后执行：
     ```bash
     chmod +x jrebel-license-active-server-*
     ./jrebel-license-active-server-*
     ```
   - 默认端口：**12345**。  
   - **注意：激活完成前不要关闭该窗口/进程。**

3. **自定义端口**（可选）  
   ```bash
   ./license-active-server --port=5555
   ```
   若需对外以 HTTPS 展示（如做索引页）：
   ```bash
   ./license-active-server --port=5555 --exportSchema=https --exportHost=jrebel.example.com
   ```

### 方式二：Docker 运行

```bash
# 拉取镜像
docker pull yuxiaoyao520/jrebel-license-active-server:latest

# 运行（映射端口 12345）
docker run -p 12345:12345 --name jrebel-license-active-server yuxiaoyao520/jrebel-license-active-server:latest
```

**docker-compose 示例：**

```yaml
services:
  jrebel-license-active-server:
    image: yuxiaoyao520/jrebel-license-active-server:latest
    container_name: jrebel-license-active-server
    ports:
      - "12345:12345"
```

### 测试/公网示例（来自官方 README）

- 在线测试地址（需手动加 `http://`）：`117.50.194.13:12345`  
- 示例激活 URL：`http://117.50.194.13:12345/524f1d03-d1d8-5e94-a099-042736d40bd9`  
- 离线天数更长的开发服：`117.50.194.13:12346`（离线约 180 天）

**建议**：自己使用请以本地运行或自建服务为主，避免依赖他人公网服务。

## 三、在 IDEA 中安装 JRebel

### 方法 1：插件市场安装（推荐）

1. 打开 IDEA → **File** → **Settings**（Windows）/ **Preferences**（macOS）。
2. 进入 **Plugins** → **Marketplace**。
3. 搜索 **JRebel** → 点击 **Install**。
4. 安装完成后重启 IDEA。

### 方法 2：从磁盘安装

1. 在 [JetBrains 插件页](https://plugins.jetbrains.com/plugin/4441-jrebel-and-xrebel/versions) 下载与当前 IDEA 版本兼容的 zip。
2. IDEA → **Settings** → **Plugins** → 齿轮图标 → **Install Plugin from Disk...**。
3. 选择下载的 zip，安装后重启 IDEA。

## 四、激活 JRebel（离线激活）

使用上面运行的 **jrebel-license-active-server** 进行激活。

### 1. 生成 GUID

- 打开 [GUID 生成](https://www.guidgen.com/) 或任意 UUID 生成器，得到一串如：`524f1d03-d1d8-5e94-a099-042736d40bd9`。

### 2. 填写激活服务器地址

- 若**本地**运行且默认端口 **12345**：
  ```text
  http://127.0.0.1:12345/你的GUID
  ```
  例如：`http://127.0.0.1:12345/524f1d03-d1d8-5e94-a099-042736d40bd9`
- 若使用**自定义端口**或**远程地址**，将 `127.0.0.1:12345` 换成实际地址与端口即可。

### 3. 在 IDEA 中激活

1. 打开 **Settings/Preferences** → 找到 **JRebel** 配置。
2. **License Server URL**：填入上一步的地址（含 GUID）。
3. **Email**：任意符合格式的邮箱（如 `your@email.com`），仅格式正确即可。
4. **重要**：
   - 激活时激活服务必须正在运行（本地运行时不要关掉命令行窗口）。
   - 此时需选择 **在线模式**（Activate online）。
5. 点击 **Activate**，看到激活成功提示即可。

### 4. 激活后切离线（可选）

- 激活成功后，在 JRebel 设置中可切换为 **Offline**，之后可关闭激活服务，在一定期限内离线使用。

## 五、使用 JRebel 热部署

### 1. 开启自动编译

- **Settings** → **Build, Execution, Deployment** → **Compiler**。
- 勾选 **Build project automatically**。

### 2. 设置自动编译间隔（可选）

- **Settings** → **Advanced Settings**。
- 设置 **Compile independent modules in parallel** 及 **Auto-compilation** 间隔（如 3–5 秒），以便保存后一段时间自动编译。

### 3. 指定要热部署的模块/包

- 在 IDEA 左下角或底部的 **Build** 区域，找到 **JRebel** 相关视图。
- 勾选需要热部署的**模块或包**，这样该包下的类在编译后会通过 JRebel 热更新。

### 4. 用 JRebel 启动应用

- 在运行配置中选择 **Run with JRebel**（或调试时 **Debug with JRebel**），启动项目。
- 修改代码并保存，等待自动编译完成，即可在不重启的前提下看到效果。

## 六、常见问题

- **激活失败**  
  确认：① 激活服务已启动；② URL 为 `http://...` 且含正确的 GUID；③ 激活时为在线模式。

- **热部署不生效**  
  检查：① 是否用 “Run with JRebel” 启动；② 是否勾选了对应模块/包；③ 是否开启了自动编译并等待编译完成。

- **端口被占用**  
  使用 `--port=其他端口` 启动激活服务，并在 License Server URL 中改用对应端口。
